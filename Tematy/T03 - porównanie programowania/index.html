Porównanie programowania obiektowego i strukturalnego

Autor: Tymoteusz Przybylski

Programowanie strukturalne

  • Moduły (podprogramy) komunikujące się przez interfejsy
  • Rozszerzenie programowania proceduralnego – lepsza czytelność i jakość kodu

Założenia programowania strukturalnego

  • Jeden punkt wejścia do bloku kodu
  • Określona kolejność wykonywania
  • Instrukcje warunkowe: if, if…else
  • Pętle: for, while, do…while
  • Unikanie goto, break, continue

Programowanie obiektowe

  • Reprezentacja problemu jako obiektów z polami i metodami
  • Bardziej naturalne dla człowieka – zgodne z rzeczywistością

Założenia programowania obiektowego

  • Abstrakcja
  • Hermetyzacja (enkapsulacja)
  • Dziedziczenie
  • Polimorfizm

Enkapsulacja

  • Dane ukryte, dostęp przez zdefiniowany interfejs
  • Settery/gettery zamiast bezpośredniego dostępu do pól

Dziedziczenie

  • Nowe klasy wywodzące się z istniejących, odziedziczające metody i pola

// Przykład w C#
class Kwadrat : Wielokat {
  public Kwadrat(int a) {
    boki = new int[4];
    boki[0] = boki[1] = boki[2] = boki[3] = a;
  }
  public override int Pole() {
    return boki[0] * boki[0];
  }
}

Polimorfizm

  • Przeciążanie/metody w różnych wariantach (np. ToString() dla różnych typów danych)

Różnice kluczowe

  • Strukturalne → funkcje/składniki układane sekwencyjnie
  • Obiektowe → metody przypisane do obiektów, mniejsza szansa na błąd przy wywoływaniu
